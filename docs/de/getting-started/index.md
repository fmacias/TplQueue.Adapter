# Einstieg

Starten Sie hier, wenn Sie TplQueue aus einer .NET- oder ASP.NET-Anwendung heraus nutzen möchten.

Dieser Bereich kombiniert die Runtime-Konzepte, die Sie aus `TplQueue.Core` benötigen, mit den adapterseitigen Integrationshinweisen aus `TplQueue.Adapter`.

## Empfohlener Pfad

1. [Installieren](install.md) Sie die öffentlichen Packages, die Sie benötigen.
2. [Erstellen Sie Ihre erste Queue](first-queue.md) aus Konfiguration oder expliziten Optionen.
3. [Bauen Sie Ihren ersten Job-Graphen](first-job-graph.md).
4. Sehen Sie sich die [öffentlichen Samples](public-samples.md) an, wenn Sie lauffähige Consumer-Flows benötigen.

## Diesen Bereich zusammen verwenden mit

- [Core-Konzepte](../core-concepts/index.md) für `IJob`, `IJobRoot`, `IDataJob` und die Retry-Selection-Regeln
- [Queues](../queues/index.md) für Queue-Verhalten und Lifecycle-Details
- [Referenz](../reference/index.md) für öffentliche API-Einstiegspunkte
- [Lizenz](../license/index.md) für die Lizenztrennung zwischen Core und Adapter
